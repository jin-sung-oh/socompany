# KAFI AI OFFICE — 프로젝트 기획/설계서 (전면 개편)

## 1. 한 줄 요약
KAFI는 **로컬 LLM 기반 데스크톱 AI 팀 비서**로, 사용자가 CEO/PM처럼 에이전트 팀을 지휘하며 업무를 수행하게 한다.

---

## 2. 문제 정의
개인 개발자는 다음 문제를 겪는다.
- 조사, 기획, 문서, 테스트가 분산되어 일관된 흐름이 끊김
- 작업 컨텍스트가 대화/노트/문서로 흩어짐
- 반복적인 리서치/요약/정리 작업이 생산성을 저해

KAFI는 **업무 흐름을 한 앱 안에 통합**하고, **역할 기반 에이전트 팀**으로 업무를 분담해 해결한다.

---

## 3. 제품 목표
- 개인 개발자가 **AI 팀과 함께 일하는 느낌**을 받도록 설계
- 빠른 피드백 루프: 지시 → 분배 → 결과 보고
- 로컬 LLM 중심: 개인정보/지적자산 유출 최소화

---

## 4. 사용자 및 사용 시나리오
### 주요 사용자
- 1인 개발자 / 프로덕트 메이커 / 스타트업 초기 팀

### 대표 시나리오
1. 사용자(CEO)가 “남성 피부관리 코치 앱 조사해줘” 요청
2. PM 에이전트가 업무를 분해하고 팀에 할당
3. Research → Planning → Document → Tester 순으로 결과 생성
4. 결과는 대시보드와 문서 패널에 요약 보고

---

## 5. 핵심 컨셉 (UX)
### 5.1 회사 시뮬레이션 UI
- 사무실 공간에 여러 동물 캐릭터가 앉아 있는 형태
- 각 캐릭터가 하나의 에이전트를 의미
- 상태는 캐릭터 동작/아이콘으로 표시

### 5.2 2개 주요 화면
- **위젯(데스크톱 위)**: 캐릭터 중심, 짧은 대화/상태 확인
- **대시보드(메인 창)**: 로그, 작업 흐름, 결과 요약, 설정

---

## 6. 에이전트 구조 (역할 기반)
### PM Agent (팀장)
- 사용자 요청 수신
- 업무 분해 및 분배
- 팀 상태/로그 관리

### Research Agent
- 정보 수집/요약
- 핵심 자료 정리

### Planning Agent
- 구조 설계
- 기능/요구사항 정의

### Document Agent
- 문서화 (Markdown 중심)
- 결과 보고서 정리

### Tester Agent
- QA 체크리스트
- 테스트 시나리오 작성

---

## 7. 상태 시스템
- 대기 / 생각 중 / 작업 중 / 완료
- 위젯/대시보드에서 캐릭터 동작과 아이콘으로 표현

---

## 8. 작업 흐름 (MVP 기준)
1. 사용자 입력
2. PM 분석 및 작업 분해
3. 에이전트별 작업 수행
4. 결과 취합 및 보고

---

## 9. 로그 시스템
- 모든 작업은 로그로 기록
- 대시보드에서 실시간 스트림 형태로 확인

예시 로그:
- PM: 신규 프로젝트 수신
- Research: 검색 및 요약 완료
- Planning: 기능 정의 완료
- Document: 문서 작성 완료

---

## 10. 기술 아키텍처 (현재 코드 기준)
### 10.1 Frontend
- Electron + React + Vite
- Zustand 상태 관리
- Three.js + React Three Fiber (3D 캐릭터)

### 10.2 Desktop (Main)
- Electron Main 프로세스
- IPC로 렌더러 ↔ 로컬 LLM 통신
- 설정/모델 목록 관리

### 10.3 Server (예정)
- `packages/server` (Fastify + Socket.IO)
- 실시간 에이전트 오케스트레이션 계획

### 10.4 LLM 엔진
- Ollama (로컬 모델 실행)
- 향후 모델 파라미터(temperature 등) 설정 추가 예정

---

## 11. 데이터 흐름
Renderer UI
→ Electron IPC
→ Local LLM (Ollama)
→ 결과 반환
→ 로그/스토어 반영

---

## 12. MVP 범위
### 포함
- 위젯 + 대시보드
- PM/Research/Planning/Document/Tester 에이전트 구조
- 로그 스트림
- Ollama 모델 선택
- 캐릭터 상태/애니메이션

### 제외
- 서버 기반 실시간 오케스트레이션
- 다중 사용자 협업
- 클라우드 배포

---

## 13. 중장기 확장
### 중기
- 대시보드 내 에이전트별 개별 채팅
- 파일/문서 컨텍스트 업로드
- 모델 파라미터 UI

### 장기
- AI 팀 협업 플랫폼
- 조직/프로젝트 단위 워크스페이스

---

## 14. 성공 지표
- 사용자가 반복 업무(조사/기획/문서화)에 걸리는 시간 30% 이상 절감
- 대시보드 내 로그/결과 확인률 80% 이상
- 위젯 이용률 (일 단위 활성)

---

## 15. 현재 작업 포인트
- 서버 패키지 실시간 오케스트레이션 연동
- 대시보드에서 에이전트별 대화 연결
- 모델 파라미터 설정 UI
