# 결정 로그

## [2026-03-01] ES 모듈 방식으로 Electron 메인 프로세스 구성
- 배경:
  - TypeScript 컴파일 후 Electron 실행 시 모듈을 찾을 수 없는 에러 발생
  - package.json에 `"type": "module"` 설정으로 ES 모듈 사용 중
- 결정:
  - import 경로에 `.js` 확장자 명시적으로 추가 (예: `./widget.js`)
  - ES 모듈에서 `__dirname` 사용을 위해 `fileURLToPath` + `dirname` 패턴 적용
- 이유:
  - ES 모듈 표준에서는 import 경로에 확장자가 필수
  - TypeScript는 `.ts` 파일을 컴파일할 때 import 경로를 변경하지 않음
  - `__dirname`은 CommonJS 전용이므로 ES 모듈에서는 `import.meta.url`로 대체 필요

- 배경:

- 이유:

  - 렌더러에서 직접 HTTP 요청보다 IPC를 통한 호출이 보안상 안전
  - SVG로 카피바라를 그려서 애니메이션과 상태 변화를 쉽게 구현

## [2026-03-01] 3D 카피바라 캐릭터 구현 및 위젯 드래그 기능
- 배경:
  - 사용자가 위젯을 움직일 수 없다는 피드백
  - 2D SVG보다 3D 캐릭터가 더 생동감 있고 재미있음
- 결정:
  - Three.js 및 React Three Fiber 사용하여 3D 카피바라 모델 구현
  - `-webkit-app-region: drag` CSS 속성으로 위젯 드래그 기능 추가
  - 상태별 3D 애니메이션 구현 (호흡, 고개 흔들기, 점프, 회전)
  - 위젯 크기를 400x450으로 조정하여 3D 씬을 보기 좋게 표시
- 이유:
  - Three.js는 웹에서 3D를 구현하는 가장 인기 있는 라이브러리
  - React Three Fiber로 React 컴포넌트처럼 3D를 선언적으로 작성 가능
  - Electron의 frameless 창에서는 `-webkit-app-region`으로 드래그 영역 지정 필요
  - 3D 모델은 상태에 따른 다양한 애니메이션 표현이 용이함

- 이유:
  - 복잡한 i18n 라이브러리(i18next 등)를 쓰지 않고도 충분히 직관적이고 가벼운 구현이 가능함
  - 설정의 영속성을 보장하기 위해 기존 `settings.json`과 `useSettingsStore`에 상태를 통합하여 관리함

  - 완전한 pathfinding이나 문 통과 규칙을 한 번에 넣기보다, 먼저 방의 존재감과 이동 제약을 체감시키는 편이 구현 리스크가 낮음
  - HUD와 로그에 구역 변화를 함께 보여주면 사용자가 현재 공간 상태를 즉시 이해할 수 있음

## [2026-03-08] 오피스 이동은 공용 navigation 레이어에서 처리
- 배경:
  - 플레이어는 이미 벽 충돌을 갖고 있었지만, 에이전트는 여전히 직선 이동이라 방 구조를 무시하고 멈추거나 겹칠 수 있었음
  - room transition, doorway pass-through, NPC 충돌을 각각 개별 컴포넌트에 덧붙이면 규칙이 쉽게 어긋날 위험이 있었음
- 결정:
  - `officeNavigation` 유틸을 추가해 clamp, wall collision, NPC personal space, nearest door, room waypoint 계산을 한 곳으로 모음
  - 플레이어 이동과 에이전트 이동 모두 같은 navigation 레이어를 사용하도록 연결
  - 문은 `roomZoneId`, `hallAnchor`, `roomAnchor`를 가진 포털 데이터로 확장하고, 플레이어는 `E/Enter`로 door anchor 사이를 통과하도록 처리
- 이유:
  - 동일한 좌표계와 이동 규칙을 공유해야 이후 proximity chat, room access, pathfinding 보정도 한 방향으로 확장 가능함
  - 에이전트가 enclosed room으로 이동할 때 최소한의 waypoint 로직만 있어도 pathfinding 없이 충분한 room transition 감각을 만들 수 있음
  - 문을 시각 장식이 아니라 포털 데이터로 바꾸면 UI 힌트와 실제 이동 결과가 일치하게 됨
