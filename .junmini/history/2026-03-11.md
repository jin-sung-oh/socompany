# 2026-03-11 작업 기록
## 완료된 작업
- `OfficeDashboard3D`를 카드형 실험실에서 실제 3D 오피스 씬으로 전면 교체
- `OFFICE_ZONES`, `OFFICE_OBJECTS`, `OFFICE_SEATS`, `useAgentStore`를 그대로 사용해 2D와 같은 좌석/이동/상태 데이터를 3D에서 공유하도록 연결
- 6종 동물 기준의 procedural 3D 사원 모델을 추가하고, 상태별 애니메이션/선택 하이라이트/이름 라벨을 구현
- `DashboardView`에서 설정값에 따라 2D 기본 오피스와 3D 커맨드 덱을 전환하도록 연결
- 설정 화면의 보기 옵션 문구를 실제 동작 기준으로 정리하고, 새 기본값을 2D 기본 오피스로 조정
- 3D 오피스에서 사원 선택 시 하단 직통 대화 패널 preview가 열리고, 기존 spatial prompt 기반으로 실제 메시지 전송/응답이 가능하도록 연결
- 공용 `OfficeConversationPanel`에 커스텀 문구 props를 추가해 2D proximity 패널과 3D 직통 채널 패널이 같은 UI를 공유하도록 정리
- `packages/server`에 FastAPI 브리지 레이어를 추가해 health check, task dispatch, webhook event sync가 가능하도록 구성
- FastAPI 실제 스펙에 맞춰 `GET /`, `GET /health`, `GET /api/v1/agents`, `POST /api/v1/tasks`, `GET /api/v1/tasks/{id}` 호출 함수를 추가
- `FASTAPI_BASE_URL`이 있을 때는 기존 백엔드 task queue가 로컬 시뮬레이션 대신 FastAPI 역할별 task 생성 + polling 흐름으로 동작하도록 전환
- FastAPI task 상태(`pending/running/completed/failed/cancelled`)를 현재 KAFI 에이전트 상태와 로그로 매핑하고, optional webhook route도 유지
- `pnpm build` 성공 확인
