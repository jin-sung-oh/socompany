# 2026-03-10 작업 기록
## 완료된 작업
- `CEO Command Center`에 선택된 실행 기준의 산출물 패널을 추가하고, `PM 초기 브리프`, `사장 보고서`, `역할별 중간 산출물`을 한 화면에서 다시 볼 수 있도록 정리
- 워크플로우 진행 흐름을 `briefing → execution → reporting → completed` 단계 카드로 시각화하고, 각 실행의 완료/오류 상태와 종료 시각을 함께 노출
- 최근 지시 8건을 다시 선택해 볼 수 있는 지시 히스토리 패널을 추가하고, 현재 실행 중인 워크플로우 상태 변경이 히스토리에 반영되도록 연결
- `WorkflowRun` 타입에 실행 식별자와 종료 시각을 추가해 결과 패널과 히스토리 카드가 같은 실행 데이터를 안정적으로 참조하도록 보강
- `@tailwindcss/vite` + `tailwindcss`를 렌더러 패키지에 실제 설치하고, Vite 플러그인 및 유틸리티 레이어를 연결해 기존 코드에 들어 있던 Tailwind 유틸 클래스가 실제로 동작하도록 전환
- Tailwind preflight는 끄고 `theme/utilities`만 먼저 도입해 기존 커스텀 CSS와 충돌을 피하면서, `SettingsPanel`, `DashboardView`, `WidgetView`의 유틸 클래스와 브랜드 컬러 유틸이 실제로 적용되도록 정리
- NPC wandering target을 전역 랜덤이 아니라 `현재 존/복도 우선`, `짧은 거리`, `문/의자 타일 회피` 기준으로 다시 뽑도록 수정하고, 비작업 상태에서 막혔을 때는 target을 비워 다시 로컬 이동을 고르도록 보정
- proximity 반응을 실제 하단 대화 패널에 연결해, 근처 NPC가 있으면 자동 미리보기 패널과 spatial voice 강도 게이지가 열리고 `대화 시작`으로 입력 모드에 바로 진입할 수 있도록 확장
- `World State` 카드와 conversation panel에 거리 기반 `Voice Channel` 라벨/볼륨 퍼센트를 노출하고, preview 패널은 닫기 후 같은 NPC에 대해 일시적으로 숨길 수 있게 dismiss 상태를 추가
- door 타일 주변의 플레이어/NPC 위치와 target을 추적해 문 오브젝트가 실제 통과 시 열리는 듯한 `doorway pass-through` 애니메이션을 갖도록 보강
- pathfinding은 동적 장애물 때문에 길이 잠시 막힐 때 `blockedPositions`를 비운 완화 경로를 한 번 더 계산해, 첫 칸이 비어 있으면 계속 전진하도록 보정
- spatial office의 벽체, 책상, 의자, 소파, 화분, 커피바, 회의 테이블에 내부 패널/엣지/소도구 레이어를 추가해 나머지 가구와 구조물도 실제 오피스 자산처럼 읽히도록 고도화
- `ui/components` 아래에 `EmptyState`, `LoadingSkeleton`, `TypingIndicator`를 추가하고, `DashboardView`, `WidgetView`, `SettingsPanel`, `OfficeConversationPanel`의 중복된 빈 상태/로딩/타이핑 UI를 공용 컴포넌트로 치환
- `App`에서 저장된 `theme` 설정을 읽어 `light/dark/system`을 실제 `data-theme`로 해석하도록 연결하고, 시스템 테마 변경도 실시간으로 반영되게 정리
- CEO 대시보드, `CEO Hotline` 위젯, spatial office HUD/대화 패널, 공용 empty/skeleton/typing UI에 dark palette override를 추가해 전체 화면이 같은 토큰 기반으로 어둡게 전환되도록 보강
- `OfficeSprites`를 추가해 바닥 타일, 벽, 문, 책상, 의자, 소파, 화분, 커피바, 회의 테이블을 16px 그리드 기준의 `crispEdges` SVG 픽셀 스프라이트로 새로 정의
- `OfficeTile`, `OfficeMap`이 기존 CSS 도형 대신 스프라이트 컴포넌트를 렌더링하도록 전환하고, 문 열림 상태와 역할별 desk accent는 유지되게 연결
- 스프라이트 전환 후에도 다크 모드에서 오피스 분위기가 유지되도록 pixel sprite 전용 필터/그림자 레이어를 추가
- 오피스 캐릭터를 종별 팔레트 기반의 chibi 실루엣으로 다시 그려, 귀/볼/배/발/꼬리와 상태 점이 보이는 더 귀여운 형태로 개선
- 플레이어와 에이전트 모두 이름 라벨, 선택/근접 하이라이트, 말풍선 위치를 새 캐릭터 비율에 맞게 조정하고 몸통/꼬리에 가벼운 생동감 애니메이션을 추가
- 이모지 얼굴은 시각 요소에서 제거하고, 종별 눈/코/수염/줄무늬/앞머리 파츠로 바꿔 오피스 캐릭터 톤을 더 통일감 있게 정리
- `pnpm build` 성공 확인
